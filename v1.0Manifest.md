# Project Current: Astro-Alchemy AI Manifest (Full Context)

## 1. Project Identity & Long-Term Vision
"Current" is a Daoist Astrology (BaZi/QMDJ) and Herbal Alchemy application. It bridges 5,000 years of clinical TCM safety with modern AI and astronomical math. 
**Ultimate Goal:** A user calculates their astrological weather, inputs their subjective "Internal Weather," and receives a highly customized, lineage-accurate, mathematically-adjusted herbal formula via a GraphRAG Python backend.
**Legal Constraint:** The app acts as an educational "General Wellness" tool, strictly avoiding medical diagnosis to comply with FDA and 2026 AI health regulations (e.g., California AB 489).

## 2. Current State & Strict Directives
We are building upon a highly functional Vue 3 MVP. 
**CRITICAL RULE:** The files in `src/core/` and `src/components/` (e.g., `QimenChart.vue`, `baziNineStar.ts`, `ganzhi.ts`) calculate complex lunar/solar data using `lunar-typescript`. DO NOT overwrite, delete, or modify this core math. Start where we are. Our immediate goal is to bind this existing math to a state manager and prep it for a backend integration.

## 3. The Target Architecture Stack
- **Frontend:** Vue 3, Vite, TypeScript, `pinia` (State), `vue-router`, `tailwindcss`.
- **Mobile Wrapper:** Capacitor (iOS/Android).
- **Backend:** FastAPI (Python 3.11+).
- **Database/Auth:** Supabase (Postgres + pgvector).
- **AI Brain:** DeepSeek API + GraphRAG (Knowledge Retrieval).

## 4. The Data Bridge (Frontend to Backend Payload)
Cursor must build the frontend state with the ultimate Python backend in mind. The Pinia store (`appStore.ts`) must cleanly serialize the following for the eventual FastAPI payload:
1. **Astro State:** Outputs of `baziNineStar.ts` and `qimen.ts` (Birth vs. Moment).
2. **User State (Internal Weather):** The existing 0-10 UI sliders (Capacity, Load, Sleep Quality, Cognitive Noise, Social Load, Emotional Tone). These represent the TCM Eight Parameters.

## 5. Backend "Far Context" (What the Python API will do)
Cursor should know what the backend will eventually handle so it formats the frontend requests properly:
- **Knowledge Graph:** Maps Herbs -> Patterns -> User State Sliders.
- **The "He Fang" Engine:** Merges and deduplicates classical formulas based on user need, adjusting dosages mathematically.
- **Safety Middleware:** A Pydantic validation interceptor that hard-blocks Tier 3 toxic herbs (e.g., Fu Zi, Ma Huang) from custom generation.
