## 3. One-Sentence Summary for AI CTO

MeridianVisualizer was refactored from hybrid HTML+SVG to 100% pure SVG, eliminating DOM vs. SVG coordinate drift; nodes are now SVG `<g>` groups with `<circle>` and `<text>` at mathematically computed vertices, with reactivity preserved for active/inactive element glow.

---

## 4. Core Logic / Architecture

- **Pure SVG container:** Single `<svg viewBox="0 0 100 100" class="w-full h-full">`; no absolute-positioned HTML divs for nodes
- **Pentagon outline:** `<polygon>` with `stroke="currentColor"` and low opacity
- **Element nodes:** Each of the Five Elements rendered as SVG `<g transform="translate(x, y)">` containing:
  - `<circle r="12">` with Tailwind `fill-*` utilities (emerald, orange, amber, slate, blue-900)
  - `<text text-anchor="middle" dominant-baseline="central" font-size="4">` for centered labels
  - `<title>` for meridian tooltip on hover
- **Reactivity:** Active elements get `opacity-100` + `drop-shadow-[0_0_8px_rgba(255,255,255,0.6)]`; inactive get `opacity-40`
- **Pentagon geometry:** Center (50, 50), radius 38, start angle -90°, step 72°. Vertices: `(cx + r·cos(θ), cy + r·sin(θ))` for θ = -90°, -18°, 54°, 126°, 198°
